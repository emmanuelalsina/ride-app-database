# Ride App Database

## Overview
Relational database design for a ride-sharing application.  
The system models users, drivers, vehicles, trips, ratings, and incident reports.

---

## Main Features
- User management (passengers and drivers)
- Role system (many-to-many relationship)
- Driver profiles and vehicle assignment
- Trip registration and tracking
- Ratings and reviews between users
- Incident reporting system

---

## Database Structure
The database includes the following main entities:

- **users**
- **roles**
- **user_roles**
- **drivers_profile**
- **vehicles**
- **trips**
- **ratings**
- **reports**

---

## Key Concepts Applied
- One-to-One relationships  
- One-to-Many relationships  
- Many-to-Many relationships  
- Foreign Keys  
- Constraints (CHECK, UNIQUE)  
- ENUM types  
- Data integrity enforcement
